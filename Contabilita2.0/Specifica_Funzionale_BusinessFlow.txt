ARCHITETTURA APPLICAZIONE

Il progetto segue un'architettura a strati con abilitazione modulare per
fasi. Le dipendenze delle funzionalita non ancora attive sono presenti
nel file requirements.txt come commenti e attivabili singolarmente.

Struttura backend (app/):
  main.py         — entry point FastAPI, middleware CORS, lifespan
  config.py       — configurazione via pydantic-settings + file .env
  database.py     — engine asincrono SQLAlchemy, session factory, init_db
  seed.py         — popolamento automatico dati demo al primo avvio
  models/         — modelli ORM SQLAlchemy
  routers/        — route FastAPI per funzionalita
  schemas/        — modelli Pydantic (request/response)
  services/       — logica di business

Struttura frontend (src/):
  app/            — pagine Next.js (App Router)
  components/     — componenti React riutilizzabili
  lib/api.ts      — client Axios con interfacce TypeScript tipizzate

Proxy API: Next.js riscrive /api/* verso http://localhost:8000/api/*
CORS backend: abilitato per http://localhost:3000

FUNZIONALITA 1 — DASHBOARD FINANZIARIA  [IMPLEMENTATA]

Descrizione: Dashboard centralizzata per monitoraggio economico e
amministrativo. Dati in tempo reale da database locale con auto-seeding
di 12 mesi di movimenti demo al primo avvio.

Funzioni implementate:
  - saldo operativo (entrate totali meno uscite totali)
  - totale entrate / totale uscite
  - cashflow netto
  - indice di liquidita (entrate / uscite)
  - grafico andamento mensile entrate/uscite/saldo (ultimi 12 mesi)
  - grafico cashflow settimanale cumulativo (ultime 8 settimane)
  - indicatore liquidita visivo con soglie cromatiche

API Endpoints (prefisso /api/v1/dashboard):
  GET /                              — risposta completa: KPI + andamento mensile + cashflow
  GET /kpi                           — solo KPI finanziari
  GET /andamento-mensile?mesi=12     — andamento mensile ultimi N mesi
  GET /cashflow-settimanale?settimane=8 — cashflow giornaliero ultimi N settimane

Modelli DB (models/financial.py):
  Conto    — id, codice, descrizione, tipo (attivo/passivo/costo/ricavo), saldo, created_at
  Movimento — id, data, tipo (entrata|uscita), importo, descrizione,
              categoria, conto_id (FK), note, created_at

Schemi Pydantic (schemas/dashboard.py):
  KPIFinanziario    — saldo_operativo, totale_entrate, totale_uscite, cashflow_netto, indice_liquidita
  SaldoMensile      — mese (YYYY-MM), entrate, uscite, saldo
  PuntoGrafico      — data, entrate, uscite, cashflow
  DashboardResponse — kpi, andamento_mensile, cashflow_settimanale, aggiornato_al
  MovimentoCreate   — data, tipo, importo, descrizione, categoria, conto_id, note
  MovimentoResponse — MovimentoCreate + id

Service (services/dashboard.py — DashboardService):
  get_kpi()                    — calcola KPI con query aggregate su Movimento
  get_andamento_mensile(mesi)  — raggruppa per mese con func.strftime
  get_cashflow_settimanale(settimane) — cashflow cumulativo giornaliero
  get_dashboard()              — aggrega i tre metodi sopra

Componenti frontend (components/dashboard/):
  KPICard             — card metrica con valore, tendenza colorata e icona
  LiquiditaIndicator  — indicatore visivo indice di liquidita
  EntrateUsciteChart  — grafico a barre mensile (Recharts, lazy-loaded SSR-safe)
  CashflowChart       — grafico cashflow settimanale (Recharts, lazy-loaded SSR-safe)

Seed dati demo (seed.py):
  - 8 categorie di entrata (vendite, servizi) e 8 categorie di uscita
  - 1-2 entrate e 1-3 uscite casuali per settimana negli ultimi 12 mesi
  - Seed deterministico (random.seed(42))
  - Idempotente: non riesegue se movimenti esistono gia

Health check: GET /health — restituisce status e versione applicazione
API docs:     http://localhost:8000/docs (Swagger UI automatico di FastAPI)

Tecnologie: Recharts, Axios, SQLAlchemy async, aiosqlite

MODALITA DI ESECUZIONE

Tipo: Web Application eseguibile in locale, accessibile via browser

Avvio rapido (Windows):
  start.bat
  Rileva automaticamente Python e Node.js, avvia backend e frontend.

Avvio manuale:
  Backend:  cd backend && uvicorn app.main:app --reload --port 8000
  Frontend: cd frontend && npm run dev

Avvio servizi opzionali con Docker (necessari per fasi F2-F8):
  docker-compose up -d
  Avvia: PostgreSQL, Redis, MinIO, Ollama

Porte:
  Frontend:        http://localhost:3000
  Backend API:     http://localhost:8000
  API docs:        http://localhost:8000/docs  (Swagger UI)
  Health check:    http://localhost:8000/health

Al primo avvio il backend crea automaticamente il database SQLite e
inserisce 12 mesi di dati demo. Nessuna configurazione manuale richiesta.

Nessun cloud, nessun server remoto, nessuna pipeline richiesta.
L'intera applicazione gira sulla macchina locale dell'utente.
